# 세대번역소 MVP 구현 계획

## 1. 목표

- 하루 안에 `세대어 번역`과 `상황별 말 바꾸기` 두 핵심 기능을 완성한다.
- 실제 OpenAI API를 연동하고 Vercel에 배포한다.
- 기능 완성, AI 결과 품질, 오류 대응, 기본 반응형 UI 순으로 우선한다.
- 3명 기준 8~10시간 내 발표 가능한 상태를 목표로 한다.

### MVP 제외 범위

- 회원가입과 인증
- 데이터베이스와 사용 이력
- 분석 이벤트와 운영 대시보드
- 응답 캐시와 스트리밍
- 관리자 화면
- Mock 또는 예시 응답 fallback

## 2. 기술 스택

| 구분 | 선택 기술 | 선택 이유 |
|---|---|---|
| 런타임 | Node.js 22, npm | 안정적인 LTS 환경과 Vercel 호환성 |
| 웹 프레임워크 | Next.js 최신 안정 버전 App Router | UI와 서버 API를 한 프로젝트에서 구현 가능 |
| 언어 | TypeScript | 클라이언트, API, AI 응답 타입을 일관되게 관리 |
| UI | React, Tailwind CSS v4 | 빠른 화면 구현과 기본 반응형 지원 |
| 서버 API | Next.js Route Handler | 별도 백엔드 없이 Vercel 서버 함수로 배포 |
| AI | OpenAI Responses API, `gpt-5-mini` | 한국어 생성 품질과 응답 속도 간 균형 |
| 검증 | Zod, OpenAI Structured Outputs | 요청과 AI 응답 구조를 동일한 스키마로 검증 |
| 테스트 | Vitest | 스키마, 서비스, API 오류 처리를 빠르게 검증 |
| 배포 | Vercel | Next.js 프로젝트의 가장 단순한 배포 경로 |

별도의 백엔드, 데이터베이스, 전역 상태 관리 라이브러리는 사용하지 않는다. 모델은 환경변수로 교체할 수 있게 구성한다.

참고 문서:

- [Next.js Route Handler](https://nextjs.org/docs/app/building-your-application/routing/router-handlers)
- [OpenAI Structured Outputs](https://platform.openai.com/docs/guides/structured-outputs?lang=javascript)
- [Tailwind CSS와 Next.js 설치](https://tailwindcss.com/docs/installation/framework-guides/nextjs)

## 3. 화면 구현

### 3.1 공통 구조

- 단일 페이지에 `세대어 번역`과 `상황별 말 바꾸기` 탭을 배치한다.
- 탭을 전환해도 각 탭의 입력과 결과 상태를 유지한다.
- 입력창에는 500자 제한, 글자 수, 전체 삭제 기능을 제공한다.
- 결과 영역은 빈 화면, 생성 중, 생성 완료, 오류 상태를 구분한다.
- 오류 상태에는 안내 문구와 재시도 버튼을 제공한다.
- 결과 복사 성공 여부를 2초 동안 표시한다.
- 모든 AI 결과에 `AI 생성` 표시와 개인·상황에 따라 표현이 다를 수 있다는 안내를 노출한다.
- 모바일 375px과 데스크톱 화면에서 핵심 기능을 사용할 수 있도록 기본 반응형을 적용한다.

### 3.2 세대어 번역 탭

- `MZ어 → 어르신어`, `어르신어 → MZ어` 방향 전환 버튼을 제공한다.
- 격식 수준을 1~5단계 슬라이더로 제공하고 기본값은 3으로 설정한다.
- 최초 결과는 번역 버튼을 눌렀을 때 생성한다.
- 결과가 있는 상태에서는 슬라이더 조작이 끝났을 때 새 격식 수준으로 자동 재번역한다.
- 번역 결과와 최대 5개의 표현 설명 카드를 표시한다.
- 표현과 설명 카드는 배열 순서에 따라 같은 색상의 배지로 연결한다.
- 설명 대상 표현이 결과 문장에 포함되면 동일 문자열을 강조한다.
- 설명할 표현이 없으면 설명 영역을 렌더링하지 않는다.

### 3.3 상황별 말 바꾸기 탭

- 교수님, 직장 상사, 회사 동료, 친구, 연인, 가족 버튼을 제공한다.
- 한 번에 하나의 대상만 선택할 수 있다.
- 입력 문장의 의도와 사실관계를 유지한 결과 한 개를 표시한다.
- 결과 복사와 오류 발생 시 재시도를 지원한다.

## 4. API 설계

### 4.1 세대어 번역

`POST /api/translate`

요청:

```json
{
  "inputText": "점메추 부탁",
  "direction": "MZ_TO_SENIOR"
}
```

성공 응답:

```json
{
  "resultText": "점심 메뉴를 추천해 주세요.",
  "termExplanations": [
    {
      "term": "점메추",
      "meaning": "점심 메뉴 추천의 줄임말",
      "nuance": "친근하고 가벼운 느낌을 주는 온라인 표현",
      "usage": "친구나 가까운 동료와 메뉴를 정할 때 사용",
      "caution": "격식 있는 자리에서는 뜻이 전달되지 않을 수 있음"
    }
  ]
}
```

### 4.2 상황별 말 바꾸기

`POST /api/rewrite`

요청:

```json
{
  "inputText": "과제 제출을 하루 늦게 해도 될까요?",
  "audience": "PROFESSOR"
}
```

성공 응답:

```json
{
  "resultText": "안녕하세요, 교수님. 부득이하게 과제 제출을 하루 늦춰도 괜찮을지 여쭙고 싶습니다. 확인해 주시면 감사하겠습니다."
}
```

### 4.3 오류 응답

```json
{
  "code": "VALIDATION_ERROR",
  "message": "입력 내용을 확인해 주세요."
}
```

| 코드 | HTTP 상태 | 의미 |
|---|---:|---|
| `VALIDATION_ERROR` | 400 | 빈 입력, 500자 초과 또는 잘못된 선택값 |
| `AI_REFUSAL` | 422 | 안전 정책상 처리할 수 없는 요청 |
| `UPSTREAM_ERROR` | 502 | OpenAI timeout, 인증, 사용량 제한 또는 서버 오류 |

### 4.4 입력 검증

- `inputText`: 앞뒤 공백 제거 후 1~500자
- `direction`: `MZ_TO_SENIOR | SENIOR_TO_MZ`
- `audience`: `PROFESSOR | BOSS | COWORKER | FRIEND | PARTNER | FAMILY`

## 5. AI 연동

### 5.1 처리 방식

- OpenAI SDK의 `responses.parse`와 `zodTextFormat`을 사용한다.
- JSON 문자열을 직접 파싱하지 않고 Zod 스키마에 맞는 구조화 응답을 받는다.
- OpenAI 클라이언트 timeout은 15초, 자동 재시도는 1회로 설정한다.
- Route Handler의 최대 실행 시간은 30초로 설정한다.
- refusal, 빈 구조화 응답, timeout과 API 오류를 구분해 공통 오류 응답으로 변환한다.
- 클라이언트에서 새 요청을 시작할 때 이전 요청을 취소해 오래된 결과가 화면을 덮어쓰지 않게 한다.
- 입력 문장과 생성 결과는 저장하거나 서버 로그에 기록하지 않는다.

### 5.2 번역 프롬프트 원칙

- 지정된 번역 방향과 격식 수준에 맞게 자연스러운 한국어 문장을 생성한다.
- 원문의 의미와 사실관계를 유지한다.
- 세대별 말투를 고정관념이나 유일한 정답으로 표현하지 않는다.
- 주요 세대 표현만 선별해 최대 5개를 설명한다.
- 각 표현에 뜻, 뉘앙스, 사용 상황과 주의 상황을 제공한다.
- 설명할 표현이 없으면 빈 배열을 반환한다.

### 5.3 상황별 변환 프롬프트 원칙

- 선택한 관계에 맞는 호칭, 인사, 어휘와 문장 구조를 사용한다.
- 필요한 경우 완곡한 표현과 맺음말을 추가한다.
- 원문의 핵심 의도와 사실관계를 유지한다.
- 원문에 없는 일정, 장소, 사유, 약속 등 구체적인 사실을 생성하지 않는다.

### 5.4 환경변수

```text
OPENAI_API_KEY=
OPENAI_MODEL=gpt-5-mini
```

API 키는 서버에서만 읽고 브라우저 번들에 포함하지 않는다.

## 6. 작업 분배 및 일정

### 0~1시간: 프로젝트 기반 구성

- 전원이 API 타입과 완료 기준을 확인한다.
- Next.js 프로젝트, 공통 타입, Tailwind CSS와 테스트 환경을 구성한다.
- Vercel 프로젝트를 연결하고 환경변수를 등록한다.

### 1~4시간: 병렬 구현

| 담당 | 작업 |
|---|---|
| 개발자 A | 공통 레이아웃, 탭, 번역 화면과 클라이언트 상태 |
| 개발자 B | OpenAI 연동, Zod 스키마, 두 API와 프롬프트 |
| 개발자 C | 상황별 화면, 공통 결과·오류·복사 컴포넌트, 테스트 fixture |

### 4~6시간: 통합

- UI와 실제 API를 연결한다.
- 요청 취소, 재시도, 표현 강조와 모바일 대응을 적용한다.
- 클라이언트와 서버 오류 형식을 일치시킨다.

### 6~8시간: 품질 조정

- 대표 문장으로 프롬프트를 검수하고 조정한다.
- 입력 검증과 오류 상황을 테스트한다.
- lint, test와 production build를 통과시킨다.

### 8~10시간: 배포 및 발표 준비

- Vercel production 환경에 배포한다.
- 모바일과 데스크톱에서 실제 API 요청을 확인한다.
- API 결제 한도와 사용 가능 상태를 확인한다.
- 발표에 사용할 입력 문장과 진행 순서를 고정하고 리허설한다.

4명 이상이면 추가 인원은 AI 결과 평가, 배포 확인과 발표 리허설을 담당한다.

## 7. 테스트 계획

### 7.1 자동 테스트

- 공백 입력, 500자와 501자 입력 검증
- 허용되지 않은 direction, audience와 formality 검증
- 정상 구조화 출력 변환
- 빈 응답과 refusal 처리
- timeout, 인증 오류, 429와 OpenAI 5xx 오류 처리
- 표현 설명이 최대 5개로 제한되는지 확인

### 7.2 수동 UI 테스트

- 탭 전환 후 입력과 결과 유지
- 번역 방향 전환
- 슬라이더 조작 완료 후 자동 재요청
- 연속 요청 중 이전 응답이 결과를 덮어쓰지 않는지 확인
- 오류 발생 후 재시도
- 결과 복사와 성공 피드백
- 모바일 375px과 데스크톱 레이아웃

### 7.3 AI 품질 검수

- 두 번역 방향에서 격식 1, 3, 5단계를 각각 확인한다.
- 교수님, 직장 상사, 회사 동료, 친구, 연인, 가족을 각각 한 문장 이상 확인한다.
- 원문의 의미와 사실이 유지되는지 확인한다.
- 원문에 없는 일정, 장소, 사유나 약속이 추가되지 않는지 확인한다.
- 특정 세대를 비하하거나 단정하는 표현이 없는지 확인한다.

## 8. 완료 기준

- 두 탭에서 실제 OpenAI API를 통한 결과 생성이 가능하다.
- 입력 검증, 로딩, 성공, 오류와 재시도 상태가 모두 동작한다.
- 결과와 표현 설명을 복사하고 확인할 수 있다.
- `npm run lint`, `npm test`, `npm run build`가 성공한다.
- Vercel production 환경에서 두 API가 정상 동작한다.
- API 키가 브라우저 번들이나 로그에 노출되지 않는다.
- 375px 모바일과 데스크톱에서 핵심 기능을 사용할 수 있다.

## 9. 전제 조건

- 구현 인원은 최소 3명이며 TypeScript로 작업할 수 있다.
- 발표 환경에서 사용할 OpenAI API 키, 결제 한도와 네트워크가 준비되어 있다.
- Mock 및 예시 fallback 없이 실제 API만 사용한다.
- 공개 장기 운영이 아닌 해커톤 심사용 단기 배포를 기준으로 한다.
- 인증, 영구 저장, 정교한 rate limit과 성공 지표 수집은 후속 작업으로 둔다.
