# 세대번역소 MVP 기획서

> 현재 구현 기준

## 1. 제품 개요

### 1.1 서비스명

**세대번역소**

### 1.2 한 문장 소개

세대번역소는 세대와 관계에 따라 달라지는 한국어 표현을 문맥에 맞게 번역해, 같은 마음이 다른 말투 때문에 오해받는 마찰을 줄이는 AI 기반 소통 도구다.

### 1.3 기획 배경

마찰은 서로 다른 두 면이 맞닿을 때 생긴다. 사람 사이에서도 서로 다른 세대와 관계가 만나면 같은 한국어를 사용하면서도 익숙한 단어와 말투가 달라 작은 마찰이 발생한다.

신조어나 인터넷 밈은 누군가에게는 자연스러운 표현이지만 다른 누군가에게는 뜻을 알 수 없는 암호가 될 수 있다. 반대로 격식 있거나 오래된 표현은 의도와 달리 딱딱하거나 거리감 있게 받아들여질 수 있다. 또한 같은 내용도 교수님, 직장 상사, 친구에게 전달할 때 필요한 말투가 각각 다르다.

세대번역소는 이 문제를 단순한 단어 치환이 아니라 **문맥과 관계의 번역 문제**로 정의한다.

### 1.4 핵심 메시지

> 우리가 없애려는 것은 세대의 차이가 아니라, 그 차이가 오해로 번지는 순간의 불필요한 마찰이다.

### 1.5 서비스 목표

- MZ세대와 중장년·노년층 사이의 표현 차이로 발생하는 오해를 줄인다.
- 낯선 밈과 MZ어를 일반적인 한국어로 풀어 이해를 돕는다.
- 중장년·노년층에게 익숙한 표현을 최근의 짧고 캐주얼한 문체로 바꾼다.
- 사용자가 전하려는 의미와 사실은 유지하면서 상대와의 관계에 맞는 문장 작성을 돕는다.
- 세대별 말투를 하나의 정답이나 고정관념으로 단정하지 않고 참고 가능한 표현을 제안한다.

### 1.6 주요 사용자

- 신조어와 인터넷 밈의 뜻을 알고 싶은 중장년·노년층
- 오래되거나 격식 있는 표현을 현재의 캐주얼한 말투로 바꾸고 싶은 청년층
- 교수님이나 직장 관계자에게 보낼 문장을 다듬고 싶은 사용자
- 가족, 연인, 친구와의 말투 차이로 의사소통에 어려움을 느끼는 사용자

## 2. 핵심 가치

### 2.1 의미 보존

표현 방식은 바꾸되 원문의 핵심 의미, 의도와 사실관계는 유지한다.

### 2.2 맥락을 고려한 번역

표면적인 단어만 치환하지 않고 문장 안에서 표현이 실제로 어떤 의미로 사용됐는지 판단한다.

### 2.3 상호 이해

어느 한 세대의 말투에 맞추도록 강요하지 않고, 서로의 표현을 이해할 수 있는 중간 지대를 만든다.

### 2.4 안전한 사용

입력 문장과 생성 결과를 저장하지 않으며, AI 결과가 개인과 상황에 따라 달라질 수 있음을 명확하게 안내한다.

## 3. 서비스 구성

서비스는 한 페이지 안의 두 개 탭으로 구성한다.

1. **세대어 번역**: MZ어와 어르신어를 양방향으로 번역하고 주요 밈·MZ어의 뜻과 예시를 제공한다.
2. **상황별 말 바꾸기**: 입력한 내용을 선택한 상대와의 관계에 알맞은 문장으로 다듬는다.

탭을 전환해도 각 탭의 입력값과 결과 상태는 유지된다.

---

## 4. 핵심 기능

### 4.1 세대어 번역

#### 번역 방향

- `MZ어 → 어르신어`: 최근 SNS의 짧고 캐주얼한 표현을 일반적으로 이해하기 쉬운 한국어로 풀어 쓴다.
- `어르신어 → MZ어`: 중장년·노년층에게 익숙한 표현을 최근 SNS에서 사용하는 짧고 캐주얼한 문체로 바꾼다.
- 화면의 전환 버튼으로 번역 방향을 바꿀 수 있다.
- 방향을 전환하면 이전 방향의 번역 결과는 초기화한다.

#### 문장 입력

- 최대 500자까지 입력할 수 있다.
- 현재 글자 수를 실시간으로 표시한다.
- 입력 내용을 한 번에 지우는 `전체 삭제` 기능을 제공한다.
- 공백만 입력하거나 500자를 초과한 요청은 처리하지 않는다.

#### 번역 결과

- 선택한 번역 방향에 맞는 결과 한 개를 표시한다.
- 원문의 사건, 감정, 요청·질문·명령과 같은 발화 의도를 유지한다.
- 결과를 클립보드에 복사할 수 있으며, 복사 성공 상태를 2초간 표시한다.
- 결과에는 `AI 생성` 표시와 참고용 결과라는 안내를 함께 제공한다.
- 생성 중에는 로딩 상태를, 실패하면 오류 안내와 `다시 시도` 버튼을 표시한다.

### 4.2 밈·MZ어 설명

번역 과정에서 확인된 주요 표현은 내장 밈 사전을 기준으로 설명한다.

- `MZ어 → 어르신어`에서는 입력 문장에서 실제 밈의 의미로 사용된 표현을 설명한다.
- `어르신어 → MZ어`에서는 번역 결과에 실제로 사용된 표현을 설명한다.
- 각 카드에는 표현의 **뜻**과 **사용 예시**를 제공한다.
- 한 번의 번역에서 최대 5개 표현을 입력 또는 결과에 등장한 순서대로 표시한다.
- 같은 단어가 음식명이나 일반 명사처럼 밈이 아닌 의미로 사용된 경우에는 설명에서 제외한다.
- 등록되지 않은 표현, 중복 표현과 문맥에 맞지 않는 표현은 서버에서 제거한다.
- 설명할 표현이 없으면 카드 영역을 표시하지 않는다.

설명 문구를 AI가 매번 새로 만드는 대신 검수 가능한 내장 사전을 사용해 뜻과 예시의 일관성을 유지한다.

### 4.3 상황별 말 바꾸기

사용자가 전달하려는 내용을 입력하면 선택한 상대와의 관계에 맞는 문장 한 개로 다듬는다.

#### 대상 선택

- 교수님
- 직장 상사
- 회사 동료
- 친구
- 연인
- 가족

#### 변환 원칙

- 관계에 맞는 호칭, 인사, 어휘와 문장 구조를 적용한다.
- 필요한 경우에만 완곡한 표현과 자연스러운 맺음말을 보완한다.
- 원문의 핵심 의미, 의도와 사실관계를 유지한다.
- 원문에 없는 일정, 장소, 사유, 약속이나 인물 등의 구체적인 사실을 만들지 않는다.
- 관계별 말투를 유일한 정답으로 단정하지 않는다.

#### 결과 기능

- 다듬은 문장 한 개를 표시한다.
- 결과 복사와 복사 성공 안내를 제공한다.
- 생성 중 상태, 오류 안내와 재시도를 지원한다.

## 5. 화면 구성과 공통 상태

### 5.1 공통 영역

- 서비스 로고와 이름, 슬로건 `말뜻과 마음 사이`
- 입력 내용을 저장하지 않는다는 개인정보 안내
- `세대어 번역`, `상황별 말 바꾸기` 탭
- AI 결과와 세대별 표현에는 개인차가 있다는 이용 안내
- 모바일과 데스크톱에서 사용할 수 있는 반응형 레이아웃

### 5.2 세대어 번역 화면

1. 번역 방향 표시와 전환 버튼
2. 번역할 문장 입력 영역
3. 번역 실행 버튼
4. 번역 결과와 복사 버튼
5. 밈·MZ어 뜻과 예시 카드

### 5.3 상황별 말 바꾸기 화면

1. 전달할 내용 입력 영역
2. 상대 선택 버튼
3. 문장 변환 실행 버튼
4. 다듬은 결과와 복사 버튼

### 5.4 공통 상태

- 입력 전 빈 상태
- AI 결과 생성 중
- 결과 생성 완료
- 입력값 검증 실패
- AI 안전 정책에 따른 처리 거절
- 외부 AI 또는 네트워크 오류

## 6. 시스템 인터페이스

### 6.1 세대어 번역 API

`POST /api/translate`

#### 요청

```json
{
  "inputText": "오늘 올린 사진 완전 느좋.",
  "direction": "MZ_TO_SENIOR"
}
```

#### 성공 응답

```json
{
  "resultText": "오늘 올린 사진은 느낌이 정말 좋습니다.",
  "termExplanations": [
    {
      "term": "느좋",
      "meaning": "느낌이 좋다.",
      "example": "오늘 올린 사진 완전 느좋."
    }
  ]
}
```

### 6.2 상황별 말 바꾸기 API

`POST /api/rewrite`

#### 요청

```json
{
  "inputText": "과제 제출을 하루 늦게 해도 될까요?",
  "audience": "PROFESSOR"
}
```

#### 성공 응답

```json
{
  "resultText": "안녕하세요, 교수님. 과제 제출을 하루 늦춰도 괜찮을지 여쭙고 싶습니다. 확인해 주시면 감사하겠습니다."
}
```

### 6.3 오류 응답

```json
{
  "code": "VALIDATION_ERROR",
  "message": "입력 내용을 확인해 주세요."
}
```

| 코드 | HTTP 상태 | 의미 |
|---|---:|---|
| `VALIDATION_ERROR` | 400 | 빈 입력, 500자 초과 또는 잘못된 선택값 |
| `AI_REFUSAL` | 422 | AI 안전 정책상 처리할 수 없는 요청 |
| `UPSTREAM_ERROR` | 502 | AI 응답, 인증, 사용량 제한 또는 외부 서버 오류 |

### 6.4 주요 데이터 타입

```text
direction:
  MZ_TO_SENIOR | SENIOR_TO_MZ

audience:
  PROFESSOR | BOSS | COWORKER | FRIEND | PARTNER | FAMILY

termExplanation:
  term | meaning | example
```

## 7. AI 및 데이터 처리 원칙

### 7.1 AI 처리 방식

- OpenAI Responses API를 사용해 번역과 문장 변환 결과를 생성한다.
- 요청과 AI 응답은 Zod 스키마와 Structured Outputs로 검증한다.
- AI는 번역문과 사전에 등록된 표현의 표제어만 반환하고, 뜻과 예시는 서버가 내장 사전에서 조회한다.
- 새 요청이 시작되면 진행 중인 이전 요청을 취소해 오래된 응답이 화면을 덮어쓰지 않게 한다.
- AI 모델은 환경변수로 변경할 수 있으며 기본값은 `gpt-5-mini`다.

### 7.2 번역 품질 원칙

- 원문의 사건, 감정, 원인과 결과를 빠뜨리지 않는다.
- 요청, 질문, 명령과 부정 등 원문의 발화 기능을 유지한다.
- 밈은 문맥에 맞을 때만 사용하며 핵심 내용을 대신하게 하지 않는다.
- MZ어 번역은 유행 표현을 과도하게 넣거나 밈을 억지로 사용하지 않는다.
- 비속어나 오해 가능성이 큰 일부 표현은 번역 결과에 사용하지 않는다.

### 7.3 개인정보 및 운영 정책

- 회원가입 없이 사용할 수 있다.
- 입력 문장과 AI 생성 결과는 영구 저장하지 않는다.
- 서버 오류 로그에는 입력 문장과 생성 결과를 포함하지 않는다.
- API 키는 서버에서만 사용하고 브라우저에 노출하지 않는다.
- 현재 MVP는 사용자 행동 분석 이벤트를 수집하지 않는다.

## 8. 품질 확인 기준

### 8.1 기능 검증

- 두 방향의 세대어 번역이 정상 동작하는지 확인한다.
- 여섯 관계에 대한 상황별 문장 변환이 가능한지 확인한다.
- 1~500자 입력과 잘못된 선택값이 올바르게 검증되는지 확인한다.
- 로딩, 성공, 오류, 재시도와 복사 상태가 정상 동작하는지 확인한다.
- 모바일과 데스크톱에서 핵심 기능을 사용할 수 있는지 확인한다.

### 8.2 번역 품질 검증

- 원문의 의미와 사실관계가 유지되는지 확인한다.
- 요청문이나 질문문이 단순 서술문으로 바뀌지 않는지 확인한다.
- 문맥에 맞지 않는 밈이 억지로 추가되지 않는지 확인한다.
- 일반 명사와 밈이 같은 표현을 문맥에 따라 구분하는지 확인한다.
- 특정 세대나 관계를 비하하거나 하나의 말투로 단정하지 않는지 확인한다.
- 상황별 변환에서 원문에 없는 구체적인 사실이 추가되지 않는지 확인한다.

### 8.3 기술 검증

- `npm run lint`
- `npm test`
- `npm run build`
- API 키와 사용자 입력이 브라우저 번들이나 오류 로그에 노출되지 않는지 확인

## 9. MVP 제외 범위

다음 기능은 현재 버전에 포함하지 않으며 후속 버전에서 검토한다.

- 회원가입과 로그인
- 번역 기록, 즐겨찾기와 사용자별 말투 학습
- 격식 강도 조절
- 사용자 직접 관계 입력
- 음성 입력과 읽어주기
- 이미지 문자 인식(OCR)
- 다국어 번역
- 사용자 참여형 세대어 사전
- 사용자 평가, 신고와 관리자 화면
- 응답 캐시와 스트리밍
- 분석 이벤트와 운영 대시보드
- 정교한 요청 횟수 제한

## 10. MVP 완료 기준

- MZ어와 어르신어의 양방향 번역이 실제 AI를 통해 동작한다.
- 번역 과정에서 확인된 밈·MZ어의 뜻과 예시가 내장 사전을 기준으로 표시된다.
- 교수님, 직장 상사, 회사 동료, 친구, 연인과 가족을 대상으로 문장을 다듬을 수 있다.
- 문장 변환 결과가 원문의 핵심 의미를 유지하고 구체적인 사실을 임의로 만들지 않는다.
- 입력 검증, 로딩, 결과, 복사, 오류와 재시도 상태가 동작한다.
- 입력 문장과 생성 결과를 저장하거나 오류 로그에 남기지 않는다.
- 모바일과 데스크톱에서 두 핵심 기능을 사용할 수 있다.
- lint, 자동 테스트와 프로덕션 빌드 검증을 통과한다.
